- **Code**: Bug fixes, new features, optimizations

- **Examples**: Real-world use cases, configuration examples

### Contribution Process
1. **Fork** the repository
2. **Create** a feature branch
3. **Make** your changes
4. **Test** your changes thoroughly
5. **Submit** a pull request

### Development Setup
```bash
# Clone your fork
git clone https://github.com/your-username/embedding-batch-workload-template.git
cd embedding-batch-workload-template

# Install dependencies
cd cdk
npm install

# Run tests
npm test

# Build the project
npm run build
```

### Code Standards
- **TypeScript**: Follow TypeScript best practices
- **Testing**: Include unit tests for new features
